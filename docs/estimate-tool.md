# 見積作成ツール MVP

`estimate.html` は、問い合わせ内容を案件区分へ整理し、見積明細とA4見積書を作る内部用ブラウザツールです。公開ページのナビゲーションには接続していません。

## 現在できること

- 問い合わせ文と条件からA〜E区分を提案
- 嵌合相手不明、安全性、短納期などの確認事項を警告
- A〜D区分の推奨作業を明細へ追加し、追加後に内容・数量・金額を編集
- 明細追加、小計・消費税・合計の自動計算
- A4印刷／PDF保存、テキストコピー
- 見積番号は発行日ごとの `YYYYMMDD_連番` 形式。印刷／PDF保存時にのみブラウザ内へ発行済み連番を記録
- PDFと編集データの初期ファイル名に見積番号を使用
- 編集データ（内部形式JSON）を保存・再読込して、作成途中の再開や過去案件の複製に利用
- 支払条件の初期値は「前払い（ご入金確認後に着手）」
- D・E区分（設計支援を含む案件）は「分割払い（着手金50％・残金50％）」を自動提案
- 支払条件は前払い・分割払い・個別設定から選択でき、見積書へ支払順序の補足も出力
- スマホでは入力欄を1列化し、見積明細をカード表示、A4プレビューを比率維持のまま縮小表示
- スクリーンショットの端末内プレビュー
- featureプレビューでは `/api/estimate` からOpenAI Responses APIへ接続し、ローカル判定と比較して任意採用

## APIテスト（featureプレビュー）

- エンドポイント：`POST /api/estimate`
- OpenAI Responses APIとJSON SchemaのStructured Outputsを使用し、`store: false`を指定
- 送信項目：問い合わせ文、主な目的、元になる情報、取付・嵌合、希望する納品物、安全性、希望時期
- 顧客名、会社名、見積番号、金額、住所、メールアドレスは独立したフォーム項目から送信しない
- 問い合わせ本文内のメールアドレス、電話番号、郵便番号、ラベル付き署名行はFunction側で外部送信前に削除
- APIは区分・確信度・理由・警告・推測目的だけを返す。料金、支払条件、明細、納品形式、見積番号は既存ロジックで決定
- 画面にはローカル判定とAPI判定を並べ、API判定は「API判定を採用する」を押した場合のみ反映
- APIキー未設定、main、本番、10秒タイムアウト、429、500系、通信失敗では `mode: local-fallback` を返す
- 顧客の問い合わせ本文やAPIキーはFunctionログへ出力しない

Cloudflare PagesのPreview環境だけに、Secret `OPENAI_API_KEY` と変数 `OPENAI_MODEL=gpt-5.6-luna` を登録します。APIキーをHTML、ブラウザ側JavaScript、GitHubへ含めないでください。本番環境への設定・反映は、Previewで一致率・応答時間・フォールバック・利用量を確認した後に行います。

### 比較テストケース

1. A：現物をスキャンしてメッシュ納品
2. B：廃番部品をCAD再構築
3. C：3Dプリント用データ作成
4. D：車両への取付部品を設計・試作
5. E：量産、販売、外注先調整を含む案件
6. 情報不足
7. 嵌合相手不明
8. 安全性不明
9. 短納期
10. 問い合わせ文と選択条件が矛盾

## 区分と基本料金

金額と分類条件は `assets/estimate.js` の `CATEGORIES` と `classifyCase()` で管理します。

- A：3Dスキャン・データ化／20,000円〜
- B：CAD再構築／60,000円〜
- C：製作用データ作成／100,000円〜
- D：設計・試作支援／200,000円〜
- E：開発・外注支援／個別見積（300,000円〜を基準表示）

## 公開前の条件

このリポジトリは静的サイトのため、ファイルを `main` へ入れるだけでは内部ツールを安全に保護できません。本番運用時は、内部ツールを別の非公開構成へ分離するか、Cloudflare Access等で認証を付けます。

OCRやAI要件抽出を追加する場合は、APIキーをブラウザのJavaScriptへ置かず、Cloudflare Worker等のサーバー側からAPIを呼び出します。画像と顧客情報の保存期間・削除方法も、その段階で決めます。
